PIMPL

pimpl全称为pointer to implementation,意思是指向实现的指针。
其优点如下：
1. 减小编译依赖(调整pimpl类后不需要重新编译)
2. 接口和实现分离(将数据和方法实现隐藏起来，只对用户开放头文件)
3. 提高可移植性(具有稳定的ABI)

PIMPL 简单实现

    //Person.h
    #include <memory>
    class Person {
        ...
    private:
        class PersonData;
        std::unique_ptr<PersonData> pData;
    };

    //Person.c
    #include "Person.h"
    #include <string>
    class PersonData {
    public:
        PersonData(const PersonData&) = delete;
        PersonData& operator=(const PersonData&) = delete;
    private:
        int age;
        std::string name;
    };

我们一般只暴露Person.h中的Person类的接口，具体的实现在PersonData类中，是对用户隐藏的。这样当我们有业务需求的更改时，只需要修改具体的实现类，而头文件不需要修改，因此不需要重新编译Person类，可以加快编译速度。同时，可以保持对外接口不变。

Qt 实现

QThread类使用到了 Pimpl，这个技术在 Qt 中非常常见，所谓的 d指针 和 q指针 就是用来实现PIMPL的

// Thread.h
class QThread : public QObject
{
    Q_OBJECT
    ...
private:
    Q_DECLARE_PRIVATE(QThread)
};

// qGetPtrHelper 为模板函数，无论d_ptr是普通指针还是智能指针都能成功返回
#define Q_DECLARE_PRIVATE(Class) \
    inline Class##Private* d_func() noexcept \
    { Q_CAST_IGNORE_ALIGN(return reinterpret_cast<Class##Private *>(qGetPtrHelper(d_ptr));) } \
    inline const Class##Private* d_func() const noexcept \
    { Q_CAST_IGNORE_ALIGN(return reinterpret_cast<const Class##Private *>(qGetPtrHelper(d_ptr));) } \
    friend class Class##Private;

不难看出Q_DECLARE_PRIVATE(QThread)展开之后其实就是提供了访问QThreadPrivate指针也就是所谓的d指针的方法d_func(),这个类也就是所谓的实现类。 这里有一个问题是d_ptr是在哪里声明的，在QThread.h中我们并没有看到，其实b_ptr是在基类QObject类中定义的。

// QObject.h
class QObject
{
    ...
protected:
    QScopedPointer<QObjectData> d_ptr; // 等同于std::unique_ptr
    ...
private:
    Q_DISABLE_COPY(QObject) // 禁止拷贝
};

QThread的 PIMPL 为 QThreadPrivate，其实还包着一层 QThreadData，这两个类都是在 qthread_p.h里声明的，但这个头文件是隐藏起来的。

在 QThread中我们一般不直接通过 d_ptr 访问数据类，我们通过一个宏定义来访问,其次，我们还有一种情况是在数据访问对外开放的类，在Qt中这两种情况分别可以说用Q_D和Q_Q宏定义来访问。

// qthread.cpp
QThread::QThread(QThreadPrivate &dd, QObject *parent)
    : QObject(dd, parent)
{
    Q_D(QThread);
    // fprintf(stderr, "QThreadData %p taken from private data for thread %p\n", d->data, this);
    d->data->thread.storeRelaxed(this);
}

//其中
#define Q_D(Class) Class * const d = d_func()

QThread类中并没有使用到q_ptrq指针，q指针的作用是访问主类(q指针指向主类)，在这里即QThread。 实际上使用q指针的方法和d指针基本一致，Q_DECLARE_PRIVATE(QThread)对应Q_DECLARE_PUBLIC(QThread),Q_D对应Q_Q,其中q_ptr在QObjectData类中声明，我们的PIMPL的基类一定是QObjectData类。

总结

  1. QT中利用d指针和q指针实现 PIMPL,任何类都可以实现，但需要保证：

    • 主类(MainClass)声明d指针
    • 主类使用宏定义 Q_DECLARE_PRIVATE(MainClass)生成访问实现类接口d_func()
    • 主类使用宏定义生成局部变量d访问实现类(MainClassPrivate)
    • 实现类声明可放在源文件也可以放在头文件中，但必须保证对外隐藏
    • 实现类(MainClassPrivate)可声明q指针访问主类
    • 实现类使用宏定义Q_DECLARE_PUBLIC(MainClass)生成访问主类接口q_func()
    • 实现类使用宏定义Q_Q生成局部变量q访问主类
    • 实现类要保证不可拷贝

  2. 主类可直接继承QObject，实现类继承QObjectData，这样不需要单独声明d_ptr和q_ptr
